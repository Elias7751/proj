## المهام

### 7.1 بناء رسالة واتساب (WhatsApp Message Builder)
- [x] إنشاء خدمة `WhatsAppService`
- [x] تنسيق رسالة الطلب بشكل مرتب وواضح
- [x] محتوى الرسالة:
  ```
  🛒 *طلب جديد #12345*
  ━━━━━━━━━━━━━━━━━━━

  📦 *المنتجات:*
  1. اسم المنتج × 2 — 5,000 ر.ي
  2. اسم المنتج × 1 — 3,000 ر.ي

  ━━━━━━━━━━━━━━━━━━━
  💰 المجموع الفرعي: 13,000 ر.ي
  🚚 رسوم التوصيل: 1,000 ر.ي
  🎫 الخصم: -500 ر.ي
  ━━━━━━━━━━━━━━━━━━━
  💵 *المجموع الكلي: 13,500 ر.ي*

  📍 *عنوان التوصيل:*
  صنعاء - شارع الزبيري - بجوار مطعم X

  👤 *بيانات العميل:*
  الاسم: أحمد محمد
  الهاتف: 777123456

  📝 *ملاحظات:*
  بدون بصل من فضلكم

  🕐 وقت الطلب: 2026/07/11 - 02:55 م
  ```

### 7.2 توليد رابط واتساب
- [x] إنشاء WhatsApp Deep Link:
  ```
  https://wa.me/967XXXXXXXXX?text=<encoded_message>
  ```
- [x] ترميز الرسالة (URL Encoding)
- [x] التعامل مع رقم هاتف المتجر (مع كود اليمن +967)

### 7.3 تدفق عملية الطلب (Order Flow)
- [x] العميل يملأ السلة
- [x] العميل يختار عنوان التوصيل
- [x] العميل يضيف ملاحظات (اختياري)
- [x] النظام يحفظ الطلب بحالة `pending_whatsapp` (تم استخدام pending)
- [x] النظام يولّد رابط واتساب
- [x] العميل يُحوَّل لواتساب تلقائياً (في الواجهة الأمامية)
- [x] صاحب المتجر يستقبل الرسالة ويؤكد/يرفض

### 7.4 تحديث حالة الطلب
- [x] صاحب المتجر يؤكد الطلب من لوحة التحكم الخاصة به
- [x] حالات الطلب:
  - `pending_whatsapp` — تم الإرسال للواتساب
  - `confirmed` — أكد صاحب المتجر
  - `preparing` — قيد التجهيز
  - `on_the_way` — في الطريق
  - `delivered` — تم التوصيل
  - `cancelled` — ملغي

### 7.5 سجل الطلبات
- [x] حفظ كل الطلبات في قاعدة البيانات
- [x] عرض سجل الطلبات للعميل
- [x] عرض الطلبات الواردة لصاحب المتجر
- [x] تتبع هل تم التواصل عبر واتساب أم لا

## الـ API Endpoints
```
# الطلب والدفع عبر واتساب
POST   /api/v1/orders/checkout               # إنشاء طلب + توليد رابط واتساب
GET    /api/v1/orders/:id/whatsapp-link      # الحصول على رابط واتساب
PUT    /api/v1/orders/:id/confirm            # تأكيد الطلب (صاحب المتجر)
PUT    /api/v1/orders/:id/status             # تحديث حالة الطلب

# سجل الطلبات
GET    /api/v1/orders/my                     # طلباتي (العميل)
GET    /api/v1/stores/:storeId/orders        # طلبات المتجر (صاحب المتجر)
GET    /api/v1/orders/:id                    # تفاصيل طلب
```

## 💡 ملاحظات مهمة

1. **لا حاجة لبوابات دفع إلكترونية** — كل شيء يتم يدوياً عبر واتساب
2. **رقم واتساب المتجر إلزامي** — يجب أن يكون لكل متجر رقم واتساب مسجل
3. **الرسالة يجب أن تكون واضحة ومنسقة** — تستخدم تنسيق واتساب (Bold, Emoji)
4. **يمكن إضافة زر "إعادة إرسال للواتساب"** — في حال لم يفتح الرابط أول مرة
5. **مستقبلاً**: يمكن إضافة WhatsApp Business API لتأكيد الطلبات تلقائياً
