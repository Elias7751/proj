# فحص نظام الإشعارات والتحليلات (Notifications & Analytics)

## ملخص النظام
1. **الإشعارات:** نظام In-App Notifications لتنبيه المستخدمين وأصحاب المتاجر بالأحداث الهامة (مثل طلب جديد، تغيير حالة الطلب).
2. **التحليلات:** يوفر إحصائيات لأصحاب المتاجر (المبيعات، الطلبات، المنتجات الأكثر مبيعاً) وللإدارة (إحصائيات المنصة ككل).

## الملفات المرتبطة
- `src/modules/notifications/notification.model.js`
- `src/modules/notifications/notification.controller.js`
- `src/modules/analytics/analytics.controller.js`

## الميزات المنفذة
- [x] إنشاء إشعار جديد (يتم استخدامه داخل الـ Controllers الأخرى مثل `order.controller.js`).
- [x] جلب إشعارات المستخدم الحالي (مقروءة وغير مقروءة).
- [x] تحديد إشعار كمقروء.
- [x] جلب ملخص تحليلات المتجر (إجمالي المبيعات، الطلبات، العملاء، المنتجات).
- [x] جلب أكثر المنتجات مبيعاً للمتجر.
- [x] جلب ملخص تحليلات المنصة للإدارة (Admin Dashboard).

## الأخطاء والمشاكل التي تم فحصها
1. **استعلامات التجميع (Aggregation Queries):**
   - تم إصلاح مشكلة أسماء الأعمدة في دوال `Sequelize.fn` و `Sequelize.col` في التحليلات، حيث يجب استخدام أسماء الأعمدة الفعلية في قاعدة البيانات (`snake_case` مثل `total_amount` و `user_id`) بدلاً من أسماء الخصائص في الـ Model (`camelCase`) لتجنب خطأ `Unknown column`.
2. **ارتباط الإشعارات:**
   - تم دمج إنشاء الإشعارات بنجاح داخل دورة حياة الطلب (عند إنشاء طلب جديد، يتم إرسال إشعار لصاحب المتجر).
