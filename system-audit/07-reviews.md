# فحص نظام التقييمات والمراجعات (Reviews System)

## ملخص النظام
نظام التقييمات يسمح للعملاء بتقييم تجربتهم بعد استلام الطلب. يمكنهم تقييم المتجر، المنتجات، أو مندوب التوصيل. كما يقوم النظام بتحديث التقييم المتوسط (Average Rating) تلقائياً للعنصر المقيَّم، ويسمح لأصحاب المتاجر بالرد على تقييمات العملاء.

## الملفات المرتبطة
- `src/modules/reviews/review.model.js`
- `src/modules/reviews/review.controller.js`
- `src/modules/reviews/review.routes.js`

## الميزات المنفذة
- [x] إضافة تقييم (نجمة 1-5 وتعليق) لمتجر، منتج، أو مندوب توصيل.
- [x] التحقق من أن الطلب مرتبط بالعميل وأن حالته `delivered` قبل السماح بالتقييم.
- [x] منع التقييم المزدوج لنفس العنصر في نفس الطلب باستخدام `uniqueIndex`.
- [x] تحديث التقييم المتوسط (`rating`) تلقائياً في جدول المتجر/المنتج/المندوب عند إضافة تقييم جديد.
- [x] جلب التقييمات الخاصة بعنصر معين مع بيانات العميل.
- [x] السماح لصاحب المتجر بالرد على تقييمات متجره أو منتجاته.

## الأخطاء والمشاكل التي تم فحصها
1. **تحديث التقييم المتوسط:**
   - الدالة `updateAverageRating` تستخدم `sequelize.fn('AVG')` لحساب المتوسط وتحديث الجدول المستهدف بشكل صحيح.
2. **التحقق من الصلاحيات والارتباط:**
   - يتم التحقق من أن المتجر المستهدف بالتقييم هو فعلاً المتجر المرتبط بالطلب.
   - يتم التحقق من أن المستخدم الذي يحاول الرد على التقييم هو فعلاً صاحب المتجر.
