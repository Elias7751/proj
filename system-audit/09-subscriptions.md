# فحص نظام الاشتراكات (Subscriptions System)

## ملخص النظام
نظام الاشتراكات يدير الباقات المتاحة لأصحاب المتاجر (مثل: مجاني، برونزي، فضي، ذهبي). كل باقة تحدد نسبة العمولة، الحد الأقصى للمنتجات، والوصول إلى ميزات متقدمة (مثل الإعلانات). يتيح النظام لأصحاب المتاجر الاشتراك في باقة وتجديدها أو إيقاف التجديد التلقائي.

## الملفات المرتبطة
- `src/modules/subscriptions/plan.model.js`
- `src/modules/subscriptions/subscription.model.js`
- `src/modules/subscriptions/subscription.controller.js`
- `src/modules/subscriptions/subscription.routes.js`

## الميزات المنفذة
- [x] إنشاء باقة جديدة (من قبل الإدارة) مع تحديد السعر الشهري/السنوي، نسبة العمولة، والميزات.
- [x] جلب الباقات النشطة لعرضها لأصحاب المتاجر.
- [x] الاشتراك في باقة (شهري أو سنوي) وحساب تاريخ الانتهاء تلقائياً.
- [x] منع الاشتراك المزدوج (إذا كان هناك اشتراك نشط بالفعل).
- [x] جلب الاشتراك الحالي للمتجر مع تفاصيل الباقة.
- [x] إيقاف التجديد التلقائي للاشتراك.

## الأخطاء والمشاكل التي تم فحصها
1. **حساب التواريخ:**
   - يتم حساب `endDate` بشكل صحيح بناءً على `billingCycle` (إضافة شهر أو سنة إلى `startDate`).
2. **التحقق من المتجر:**
   - يتم جلب المتجر المرتبط بالمستخدم الحالي لضمان عدم اشتراكه نيابة عن متجر آخر (إلا إذا كان أدمن).
3. **حالة الدفع:**
   - حالياً يتم تفعيل الاشتراك مباشرة `status: 'active'`. في بيئة الإنتاج الحقيقية، يجب أن يكون `pending_payment` حتى يتم تأكيد الدفع.
