# فحص نظام العروض والإعلانات (Offers & Ads System)

## ملخص النظام
يتكون هذا النظام من جزأين:
1. **العروض والكوبونات:** يتيح لأصحاب المتاجر إنشاء عروض ترويجية على منتجات محددة، وإنشاء كوبونات خصم (نسبة مئوية أو مبلغ ثابت) مع شروط استخدام.
2. **الإعلانات:** يتيح لأصحاب المتاجر طلب مساحات إعلانية في المنصة (الصفحة الرئيسية، داخل التصنيفات، إلخ)، وتقوم الإدارة بالموافقة عليها. يقوم النظام بتتبع المشاهدات والنقرات.

## الميزات المنفذة
- [x] إنشاء عرض ترويجي وربطه بمنتجات محددة.
- [x] جلب العروض النشطة (التي تقع ضمن فترة الصلاحية).
- [x] إنشاء كوبون خصم مع شروط (الحد الأدنى للطلب، الحد الأقصى للخصم، حد الاستخدام).
- [x] التحقق من صلاحية الكوبون وحساب قيمة الخصم.
- [x] طلب إعلان جديد وحساب التكلفة التقديرية بناءً على الأيام.
- [x] جلب الإعلانات حسب الموضع (Placement) وتسجيل المشاهدات تلقائياً.
- [x] تسجيل النقرات على الإعلانات.
- [x] موافقة/رفض الإعلانات من قبل الإدارة.

## الأخطاء والمشاكل التي تم فحصها
1. **التحقق من الكوبون:**
   - يتم التحقق من تاريخ الصلاحية، حالة الكوبون، حد الاستخدام، والحد الأدنى للطلب بشكل صحيح.
   - يتم حساب الخصم بشكل دقيق (سواء كان نسبة مئوية مع حد أقصى، أو مبلغ ثابت).
2. **تتبع الإعلانات:**
   - يتم استخدام `increment` في Sequelize لزيادة عداد المشاهدات والنقرات بشكل آمن (Atomic Updates).
3. **التواريخ:**
   - يتم استخدام `Op.lte` و `Op.gte` لمقارنة التواريخ وجلب العروض والإعلانات النشطة فقط.
