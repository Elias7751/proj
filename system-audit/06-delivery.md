# فحص نظام التوصيل (Delivery System)

## ملخص النظام
نظام التوصيل يتيح للمستخدمين التسجيل كمندوبي توصيل، ويتيح لأصحاب المتاجر أو الإدارة تعيين الطلبات للمندوبين. كما يتيح للمندوبين تحديث حالتهم (متاح/مشغول)، موقعهم، وحالة التوصيلة (تم الاستلام، في الطريق، تم التوصيل).

## الملفات المرتبطة
- `src/modules/delivery/driver.model.js`
- `src/modules/delivery/delivery.model.js`
- `src/modules/delivery/delivery.controller.js`
- `src/modules/delivery/delivery.routes.js`

## الميزات المنفذة
- [x] تسجيل مستخدم كمندوب توصيل (يتحول دوره تلقائياً إلى `delivery` وتكون حالته `pending_approval`).
- [x] تحديث حالة المندوب (متاح، مشغول، غير متصل).
- [x] تحديث موقع المندوب (خطوط الطول والعرض).
- [x] تعيين طلب لمندوب متاح (من قبل صاحب المتجر أو الإدارة).
- [x] تحديث حالة التوصيلة من قبل المندوب (تحديث حالة الطلب المرتبط تلقائياً).
- [x] جلب قائمة التوصيلات الخاصة بالمندوب.

## الأخطاء والمشاكل التي تم فحصها
1. **تحديث حالة الطلب تلقائياً:**
   - عند قيام المندوب بتحديث حالة التوصيلة إلى `picked_up`، تتغير حالة الطلب إلى `on_the_way`.
   - عند التحديث إلى `delivered`، تتغير حالة الطلب إلى `delivered` ويتم تسجيل وقت التوصيل الفعلي.
2. **تحديث حالة المندوب تلقائياً:**
   - عند قبول توصيلة، يتحول المندوب إلى `busy`.
   - عند إتمام التوصيلة أو إلغائها، يعود المندوب إلى `available`.
3. **الصلاحيات (Authorization):**
   - لا يمكن للمندوب تحديث حالته إذا كان حسابه لا يزال قيد المراجعة `pending_approval`.
   - لا يمكن تعيين طلب لمندوب غير متاح `available`.
