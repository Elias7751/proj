# فحص نظام المتاجر (Stores System)

## ملخص النظام
نظام المتاجر يسمح لأصحاب المتاجر بإنشاء متاجرهم الخاصة، وإدارتها، وربطها بالتصنيفات والمدن. كما يسمح للعملاء بتصفح المتاجر والبحث فيها.

## الملفات المرتبطة
- `src/modules/stores/store.model.js`
- `src/modules/stores/store.controller.js`
- `src/modules/stores/store.routes.js`

## الميزات المنفذة
- [x] إنشاء متجر جديد (يتم تعيين الحالة إلى `pending` لأصحاب المتاجر، و `active` للمسؤولين).
- [x] توليد `slug` تلقائي بناءً على الاسم باللغة الإنجليزية.
- [x] جلب المتاجر النشطة مع دعم البحث (بالاسم أو الوصف) والفلترة (حسب التصنيف أو المدينة).
- [x] جلب بيانات متجر محدد باستخدام الـ `slug`.
- [x] تعديل بيانات المتجر (صلاحية للمالك أو المسؤول).

## الأخطاء والمشاكل التي تم فحصها
1. **الصلاحيات (Authorization):**
   - يتم التحقق من أن المستخدم لديه دور `store_owner` أو `admin` قبل إنشاء المتجر.
   - يتم التحقق من أن المستخدم هو مالك المتجر أو `admin` قبل التعديل.
2. **التحقق من البيانات المرتبطة:**
   - يتم التحقق من وجود التصنيف (`categoryId`) والمدينة (`cityId`) والمنطقة (`areaId`) قبل إنشاء المتجر لتجنب أخطاء قاعدة البيانات.
3. **تكرار الـ Slug:**
   - يتم التحقق من عدم وجود متجر آخر بنفس الـ `slug`.
