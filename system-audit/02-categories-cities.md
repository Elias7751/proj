# فحص نظام المتاجر والتصنيفات (الجزء الأول: التصنيفات والمدن)

## ملخص النظام
نظام التصنيفات يدير هيكلة المنتجات (تصنيفات رئيسية وفرعية)، بينما نظام المدن والمناطق يدير النطاق الجغرافي لعمل المنصة وتحديد رسوم التوصيل بناءً على المنطقة.

## الميزات المنفذة
- [x] إنشاء تصنيف رئيسي أو فرعي مع توليد `slug` تلقائي.
- [x] جلب التصنيفات النشطة فقط مع تصنيفاتها الفرعية.
- [x] إنشاء مدينة جديدة.
- [x] إضافة مناطق للمدينة مع تحديد رسوم التوصيل الخاصة بكل منطقة.
- [x] جلب المدن النشطة مع مناطقها النشطة.

## الأخطاء والمشاكل التي تم فحصها
1. **توليد الـ Slug:**
   - الدالة المساعدة `slugify` تدعم الحروف العربية والإنجليزية وتزيل المسافات والرموز غير المرغوب فيها بشكل صحيح.
2. **العلاقات (Associations):**
   - علاقة التصنيف بنفسه (Self-referencing) لإنشاء تصنيفات فرعية تعمل بشكل صحيح (`parentId`).
   - علاقة المدينة بالمنطقة (One-to-Many) تعمل بشكل صحيح.
3. **التحقق (Validation):**
   - يتم التحقق من عدم تكرار الـ `slug` للتصنيف.
   - يتم التحقق من وجود المدينة قبل إضافة منطقة لها.
